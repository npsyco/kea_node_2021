Basic server setup
  • Here we see the fundamental setup of a node server: we need to instruct node to utilize express, our web framework for node.js
  • The app.use(express.static('public')) instructs node to include a folder called public (named so by convention). This will then enable us to keep our html-files (and other static files such as css-files or images) in a specific folder (or subfolder).
  • The app.get serves the pages, where the "/" is the endpoint and the (req, res) is the parameters for request and response. This is followed by res.redirect on line 9, so user always defaults to home.
  • The response.sendFile is where an actual html file is served.

Server port

  • On line 43, we see that the port is set to the port that is = process.env.PORT. This is set in the project files (package.json). However, if this does not exists, the app will launch on port 8000. Then the app.listen is what instruct our server to utilize a specific port. In this case we supplied it with the parameter of port, instead of an actual number. If there are no errors, we will have a console print with the port specified in the .json file.
